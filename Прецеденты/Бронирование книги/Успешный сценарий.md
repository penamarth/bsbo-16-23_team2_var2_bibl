Аутентификация, проверка на долги, поиск и бронирование книги

Процесс:
1. Аутентификация на сайте (класс ISB). Пользователь предоставляет qr-code по которому библиотекарь проверяет данные аккаунта (ScanQRCode).
2. Проверка на наличие (аккаунта) (TryFindAccount) и на долги(не сдал вовремя книгу). (Класс Account) (HasOverdueBooks, HasUnpaidFines, CanBorrowMore)
3. Если всё хорошо, то библиотекарь ищет в библиотеке книгу для пользователя. (вызывается поиск по паттерну, начиная с ISB)
4. Записывает в очередь. (класс ISB, ReserveBook)
5. Возвращаем позицию книги и библиотекарь выдаёт книгу пользователю.

```
@startuml
actor Librarian
participant "ISB" as ISB
participant "Account" as Account
participant "Catalog" as Catalog
participant "Component" as Component
participant "ReservationQueue" as Queue
participant "Reservation" as Reservation

== Book Reservation ==

Librarian -> ISB: scanQRCode(qrData)
activate ISB

ISB -> ISB: tryFindAccount(id)

ISB -> ISB: getUserInfo(id)

ISB -> Account: hasOverdueBooks()
activate Account
Account --> ISB: false


ISB -> Account: hasUnpaidFines()

Account --> ISB: false


ISB -> Account: canBorrowMore()

Account --> ISB: true
deactivate Account

ISB --> Librarian: accountValidated

/' если книга есть то бронируем '/

Librarian -> ISB: findBooks(title, author)

ISB -> Catalog: findBooks(title, author)
activate Catalog
Catalog -> Component: findBooks(title, author)
activate Component
Component --> Catalog: bookList
deactivate Component
Catalog --> ISB: searchResults
deactivate Catalog
ISB --> Librarian: showResults


Librarian -> ISB: reserveBook(bookID)
ISB -> Queue: addReservation(bookID,accountID)
activate Queue
Queue -> Reservation: addReservation(reservation: Reservation)
activate Reservation
Reservation -> Queue: None
deactivate Reservation
Queue -> Queue: getPosition(bookId)
Queue --> ISB: position
deactivate Queue


ISB --> Librarian: "Book reserved."
deactivate ISB
@enduml

```
